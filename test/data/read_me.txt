База данных реляционного типа, предназначенная для хранения данных, предназначенных только для чтения.

Содержит единственный генератор идентификаторов, словарь данных, таблицы с данными.

Используется в качестве промежуточной формы хранения для модулей.
Может использоваться для хранения данных пользователя
    Количество строк втаблицах не меняется, 
    Максимаьное коичество строк в каждой таблице задается утилитой администрирования
    Возможна поддержка индексов во внешнем файле

Блоки данных с выравниванием по границе 2048 байт

Структура описания баз:

    "PLDATA  "   C(8) - мнемоника иденификатора базы PL2
    "01"         C(2) - мнемоника версии формата
    reserved     B(21) - пространство с символами пробела
    indexed      L(1) - признак использования одноименного файла с индексами
    schema_count H(4) - количество описаний схем
    schema_start H(8) - начало описания схем в потоке
    key_start    H(8) - начало описания вторичных ключей
    last_id      H(8) - последний использованный идентификатор

Описание схем данных:

    schema_name  S(64) - имя схемы данных 
    code_page    H(4) - кодовая страница по умолчанию 
    table_count  H(4) - количество таблиц
    schema_start   H(8) - смещение начала описания схемы данных в потоке


Описание таблицы:

    table_name   S(64) - название таблицы
    field_count  H(8) - количество аттрибутов
    record_count H(8) - количество записей
    field_start  H(8) - смещение начала описания полей в потоке
    data_start   H(8) - смещение начала данных табицы в потоке

Описание полей:

    field_name   S(64) - имя поля
    field_type   S(2) - тип поля
    field_size   H(4) - длина данных поля
    field_index  H(2) - маска индексов для поля
        0 - индекс не нужен
        1 - индекс по полю
        2 - нидекс по верхнему регистру (поиск без учета регистра)
        4 - входит в составные индексы

Типы данных для field_type:

    H - шестнадцатеричная запись целых чисел
    N - десятичная запись чисел
    C - однобайтовые строки в указанной кодировке
    S - двухбайтовые строки UTF16
    U - строки в кодировке UTF32
    D - дата в формате YYYYMMDD
    T - время в формате HHMMCCXXXXXX с десятичными долями секунд с указанной точностью
    L - логическая величина с пятью состояниями:
        T - точная истина
        t - предположительная истина
        S - неопределенное состояние
        f - предположительная ложь
        F - точная ложь
    B - цепочка битов
    R - бинарные данные указанной длины в байтах 
        каталог:

            start H(8) - ссылка на смещение начала следующего элемента каталога - четыре байта
            - элементы каталогах[85]:
               id    H(8) - идентификатор данных
               start H(8) - смещение начала элемента данных - четыре байта
               size  H(8) - длина двоичных данных - четыре байта 
        двоичные данные

Описание вторичных ключей:

    schema       S(64) - схема для вторичного ключа
    table        S(64) - таблица для вторичного ключа
    field        S(64) - поле для вторичного ключа
    link_schema  S(64) - схема с первичным ключом
    link_table   S(64) - таблица с первичным ключом 
    link_field   S(64) - поле с первичным ключом
